# 提交规范（Git Commit / Push 规则）

> 读者：在本仓库（AINews）工作的 AI agent 与维护者。
> 约定：**以后所有提交、推送都按本文件执行。** 与本文件冲突的临时口头指令，需先确认。

---

## 0. 一句话流程

```bash
# 1 只暂存本任务相关文件（核对！）  2 规范 message 提交  3 走 gh HTTPS 推 main
git add <仅本任务文件>
git status --short                       # 核对：只有该改的，没有误带
git commit -m "type(scope): 中文简述 ..."
gh auth setup-git && git push origin main
```

---

## 1. 推送通道（硬约束）
- **必须走 gh HTTPS 凭证**：本机 SSH key（luwenyiCC）对本仓库**无写权限**，直接 SSH push 会失败。
- 远端已是 HTTPS：`https://github.com/boomlulu/ai-news.git`。
- 推送前执行 `gh auth setup-git`（配置 gh 为 git 凭证助手，幂等），再 `git push origin main`。
- gh 账号需为 `boomlulu` 且有 `repo` 权限（`gh auth status` 可查）。

## 2. 分支策略
- **直接提交到 `main`**。本仓库是 GitHub Pages 从 `main` 部署的个人站点，**无 PR / 无功能分支流程**。
- 不要为常规改动新建分支（与既有工作流和部署冲突）。

## 3. 提交节奏
- **每完成一个独立步骤就 commit + push 一次**，不囤积、不憋大提交。
- 一个提交只做一件事，便于回滚与 review。

## 4. 提交粒度（防误带）
- **只暂存与当前任务相关的文件**。**禁止**习惯性 `git add -A` / `git add .` 把工作区里无关改动一并带上。
- 提交前必跑 `git status --short` 核对暂存清单 = 预期清单。
- 工作区若有其他人/其他任务的未完成改动，绕开它们，别打包。

## 5. Commit message 格式
- 首行：`type(scope): 中文简述`
  - `type` ∈ `feat` | `fix` | `chore` | `docs` | `refactor` | `style` | `perf`
  - `scope` 用模块名，如 `tts`、`template`、`archive`（可省略）
  - 简述用**中文**，一句话讲清"做了什么"
- 需要细节时：首行后空一行，再写要点列表（每条一句）。
- **结尾固定加 trailer**（空一行后）：
  ```
  Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
  ```
- 示例：
  ```
  docs(tts): 提交规范补充 .gitignore 约定（产物入库口径）

  - 明确 tts/samples 下试听/实验件不入库、每日正式口播 mp3 入库
  - 与引擎迁出 AISpeak 后的本仓库现状对齐

  Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
  ```

## 6. 提交前验证（别把坏东西提交）
- **配置改动**：JSON/YAML 先 validate（如 `python -c "import json;json.load(open('tts/config.json'))"`）。
- **代码改动**：关键路径能跑通再提交。
- **音频产物**：用 whisper 转写产物对照原稿，确认内容真对——**不要只看"没报错/有文件"**（曾出现 0 报错但全乱码）。
- **文档/数据**：自检无本机绝对路径泄漏等。

## 7. .gitignore 约定（什么不入库）
- **TTS 引擎已迁出本仓库到 `/Users/boom/Demo/AISpeak`**：引擎重型件（`.venv`、`models`、`vendor` 等）与生成代码不再在 AINews，相关忽略规则也随之移走，本仓库不再维护。
- AINews 仅保留 `tts/samples/`（每日口播产物存档）。其中草稿/试听件**不入库**：试听件（`tts/samples/*.wav`）、A-B 实验件（`tts/samples/_pacing_*`、`tts/samples/daily-ai-news-*-broadcast-myvoice*.mp3`）、voicetest、`tts/samples/_excerpt.txt`、`*-macsay.mp3`；另外 `Template/*.m4a` 等也不入库。
- **每日正式口播 mp3 仍入库**：由 07:00 autopush 自动提交，作为产物存档。
- 新增大二进制/可重生成产物前，先想清楚该不该进 git 历史（进了难撤销）。

## 8. 不可逆 / 对外操作 先确认
- `push` 即对外发布（GitHub 公开、Pages 上线、可能被缓存/索引）；改写历史、`reset --hard`、删文件等——执行前确认。
- 要删除/覆盖的目标**先看清内容**：与描述不符、或非本会话所建，先反馈再说，别闷头动手。

---

## 9. 速查
| 场景 | 命令 |
|---|---|
| 看待提交 | `git status --short` |
| 暂存指定文件 | `git add tts/config.json` |
| 强制入库被忽略的正式 prompt | 在 .gitignore 加 `!路径` 后 `git add` |
| 提交 | `git commit -m "docs(tts): ..."`（含 Co-Authored-By） |
| 配 HTTPS 凭证 | `gh auth setup-git` |
| 推送 | `git push origin main` |
| 确认同步 | `git status -sb`（`main...origin/main` 无 ahead/behind） |
