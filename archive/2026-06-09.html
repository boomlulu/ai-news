OpenAI 称「聊天已死」，ChatGPT 将重写为 Agent 超级应用

来源：金融时报 · TechCrunch · The Decoder2026-06-08可信度：中高
Lead1 / 3

OpenAI 内部称「聊天已死」，启动 ChatGPT 上线以来最大改版，将其重写为以 Agent 为核心的「超级应用」。

据金融时报，新版拟合并 ChatGPT、Codex 编程平台与 Atlas 浏览器三条产品线，接入 Canva、Booking.com 等合作方应用；公司已内部重组，把 ChatGPT、Codex 与产品团队统一交由新任首席产品官 Thibault Sottiaux。企业用户已占其营收 40%，目标年底升至 50%，改版正值其筹备年内 IPO。

IMPACT 9.1
Analysis2 / 3

看点：头部实验室第一次公开宣判「对话框」范式终结——竞争主轴从「谁答得好」转向「谁能替你把事办完」，也是 OpenAI 给 IPO 投资人「ChatGPT 不止是聊天玩具」的答案。

  • 产品形态：聊天框让位给 Agent 工作流，编程、生图、第三方应用被编排进同一入口。
  • 竞争格局：与 Anthropic（Claude Code）、谷歌（Gemini Agent）的「Agent 平台」之争全面摊牌。
  • 商业逻辑：押注企业付费撑起 IPO 估值叙事，消费级「答问」不再是核心增长点。

可信度中高
待验证新版界面与「超级应用」尚未上线（官方称未来数周推出），合作方清单与 Atlas 整合细节待落地。
建议动作构建在 OpenAI 生态上的团队应提前评估 Agent 化入口与 Codex 合并对接口、计费的影响。

Perplexity 推「搜索即代码」，让模型自己写检索管线

来源：Perplexity Research · The Decoder · Winbuzzer2026-06-07可信度：高
Lead1 / 3

Perplexity 发布「Search as Code」，让 AI Agent 自行生成 Python 脚本来调度搜索，取代固定的搜索 API。

架构由推理模型作「控制面」拆解任务并生成代码、安全沙箱执行、模块化 Agentic Search SDK 三部分组成；模型可自建排序、网格搜索与 map-reduce 管线。官方称在一项复杂 CVE 研究任务上，token 用量较标准管线下降 85%。该能力已并入 Agent API，并成为其 Agent 平台 Computer 的默认搜索方式。

IMPACT 7.7
Analysis2 / 3

看点：检索从「调用固定 API」升级为「模型现写现跑代码」，是 Agent 把外部工具内化为可编程能力的方向标，也直指 RAG/搜索接口的下一代形态。

  • 开发者：复杂检索任务可显著省 token、降成本，长链路任务不易「跑偏」。
  • 搜索/RAG 厂商：固定 API 范式受挑战，「让模型自写管线」可能成新基线。
  • 安全：模型自动执行代码须依赖沙箱隔离，注入与越权风险成新焦点。

主要来源Perplexity Research · The Decoder · Winbuzzer
可信度高
待验证85% 省 token 为官方单任务数据，跨场景表现与沙箱安全性需第三方验证。
建议动作做检索密集型 Agent 的团队可在沙箱环境试评估「代码化检索」对成本与质量的影响。

安全Page 03 · Impact 7.9

ChatGPT 上线「锁定模式」，断网防提示注入窃密

来源：OpenAI · TechCrunch · The Decoder2026-06-06可信度：高
Lead1 / 3

OpenAI 为 ChatGPT 推出可选的「锁定模式（Lockdown Mode）」，通过限制对外网络请求来降低提示注入导致的数据外泄风险。

开启后将停用实时联网浏览（仅访问缓存）、联网取图、深度研究与 Agent 模式，且不能下载文件做数据分析；面向处理敏感数据的个人与组织，在「设置 > 安全」中开启。OpenAI 明确警告：即便开启，缓存内容或上传文件中的注入仍可能影响回答。

IMPACT 7.9
Analysis2 / 3

看点：当 Agent 被允许联网与操作工具，提示注入就从「理论风险」变成「会偷数据的真攻击」——头部厂商把「主动断网」做成产品开关，等于承认现有防护挡不住，先给高敏用户兜底。

  • 企业：处理敏感数据的团队多了一个「以功能换安全」的合规开关。
  • 行业：把提示注入风险产品化、显性化，或推动同行跟进类似「安全档位」。
  • 局限：官方承认锁定模式不能根治注入，安全责任仍在使用方流程设计。
Sources3 / 3
主要来源OpenAI · TechCrunch · The Decoder
可信度高
待验证锁定模式实际防护强度与对正常工作流的影响，需安全团队实测评估。
建议动作在 ChatGPT 处理敏感数据的组织可评估对高风险账号默认开启锁定模式。

自称 OpenAI 自研芯片项目「第二位硬件员工」的 Clive Chan 离职加入 Anthropic，正值两家公司同步冲刺 IPO。

Chan 此前在特斯拉 Autopilot 做 ML 训练定制芯片，2024 年 1 月加入 OpenAI，参与从零搭建定制芯片并涉及 OpenAI–博通合作。Anthropic 据报道正考虑自研 AI 芯片（截至 4 月仍处早期、尚无专职团队）；其加盟是用于自研芯片还是优化现有硬件软件，目前尚不明确。
