# 语音转文字（ASR / Speech-to-Text）—— Agent 操作指南

> 读者：后续接手的 AI agent。
> 目的：把一段**音频/视频**转成**文字稿**（纯文本 / 带时间戳 / 词级时间戳 / SRT 字幕）。
> 原则：**选对模型档位 → 跑 → 用客观方式验收**，别拿「没报错 / 有输出」冒充「转对了」。
> 本文命令均在仓库根 `/Users/boom/Demo/AINews` 下执行，Python 一律用隔离环境 `tts/.venv/bin/python`。

---

## 0. 结论先行（TL;DR）

1. **当前只有一个引擎**：`openai-whisper`（OpenAI Whisper，本地 CPU 跑）。本文第 5 节预留了**扩展口**，后续接 faster-whisper / FunASR / 云端 API 时**业务侧不用改**。
2. **默认档位用 `small`**：质量/速度平衡点。要快/只探测用 `base`；要精确（专有名词、做字幕）用 `medium`。
3. **中文务必带三件套**：`language="zh"` + `fp16=False`（CPU 必须）+ `initial_prompt`（喂专有名词，**强制简体**、纠正公司名、补标点，提升巨大）。
4. **本机别开 MPS(GPU)**：openai-whisper 在 Apple Silicon 上 MPS 反而更慢（实测见第 2 节），老老实实 CPU。
5. **长音频不要一把梭**：先确认时长；几分钟可直接转，几十分钟以上考虑切段或换 faster-whisper（第 5 节）。
6. **验收**：抽查关键词/专有名词是否命中、句子是否通顺。乱码（中英混杂、人名全错）= 没转对，重跑或升档 + 加 `initial_prompt`。

## 2. 模型档位 · 速度 · 质量横评（本机实测）

> 基准条件：**Apple M5 Pro（15 核）· CPU · openai-whisper 20250625 · 30s 中文播报音频 · `language="zh", fp16=False`**。
> `xRT` = 音频时长 ÷ 转写耗时（越大越快；39× 表示 1 秒音频约 0.026 秒转完）。**速度随机器/音频内容浮动，换机重测。**

| 模型 | 参数量 | 权重 | 加载 | 转写 30s | **xRT(越大越快)** | 中文质量 |
|---|---|---|---|---|---|---|
| `tiny`  | 39M  | ~75M  | — | —（未测） | 最快 | 探测都嫌糙，不建议中文 |
| `base`  | 74M  | 139M  | 0.30s | **0.77s** | **39×** | ✗ 差：专名全错、中英混乱 |
| `small` | 244M | 461M  | 1.09s | **1.80s** | **16.7×** | ◯ 中：大意对，专名偶错（**默认档**） |
| `medium`| 769M | 1.4G  | 3.09s | **5.48s** | **5.5×** | ✓ 好：专名基本对，偶混繁体 |
| `large-v3` | 1550M | ~2.9G | —（未缓存） | 预计 ~2–3× | 预计 < 实时附近 | ✓✓ 最好，但 CPU 慢、占内存大 |

**MPS(GPU) 加速实测：不划算。** medium 在 MPS 上转写 6.40s，比 CPU 的 5.48s **更慢**——openai-whisper 很多算子在 MPS 上回退 CPU，反而吃调度开销。本机一律 CPU。要 GPU 提速请走 faster-whisper（第 5 节扩展口）。

### 同一句话的转写结果横向对比（直观看质量差）

原句大意：「Anthropic 估值 9650 亿美元反超 OpenAI 并申请 IPO」

| 配置 | 转写结果（前段） | 判定 |
|---|---|---|
| `base` | `3.And for a pic,估值9650億美元, 反超 Open` | ✗ Anthropic→「And for a pic」，中英乱 |
| `small`（无 prompt） | `3. Antfrappek 估值9,650亿美元反超Ocon AI` | ◯ 大意对，Anthropic/OpenAI 仍错 |
| `medium`（无 prompt） | `3. Anthropic 估值9650億美元反超 Oken AI 並` | ✓ Anthropic 对，OpenAI 仍错 + 混繁体 |
| **`small` + `initial_prompt`** | `3. Anthropic 估值9,650亿美元。反超OpenAI，并申请IPO。` | ✓✓ 专名/标点/简体全对 |

**结论：`small` + 一句 `initial_prompt`，性价比打过裸跑 `medium`。** 这是中文转写的首选组合。

## 3. 关键参数速查（`whisper` 的 `transcribe()`）

| 参数 | 推荐值 | 说明 |
|---|---|---|
| `language` | `"zh"` | **务必指定**，省掉语种探测、避免误判成别的语言 |
| `fp16` | `False` | **CPU 必须 False**，否则报警告/变慢；GPU 才用 True |
| `initial_prompt` | 一句领域提示 | **中文最重要的旋钮**：列出会出现的公司/人名/术语、写明「简体中文普通话」，强制简体 + 纠专名 + 补标点 |
| `word_timestamps` | `True`/`False` | 要词级时间戳（做精切、卡拉OK字幕）才开；慢一点 |
| `task` | `"transcribe"` / `"translate"` | `translate` = 直接转成英文（很少用） |
| `temperature` | `0`（默认带回退） | 设 `0` 更稳定；默认是 `(0,0.2,...,1.0)` 失败回退链 |
| `condition_on_previous_text` | `True`（默认） | 长音频出现「复读机」式重复时，设 `False` 切断上文依赖 |
| `verbose` | `False`/`None` | `True` 会逐段打印（调试用） |

**`initial_prompt` 模板（直接抄改）：**
```
以下是简体中文普通话内容。涉及公司与术语：Anthropic、OpenAI、谷歌、英伟达、Claude、GPT、大模型、估值、IPO。
```

## 4. ✅ 成功路径（照抄即可）

### Step 0 — 探测源（时长 / 是否要先抽音轨）
```bash
SRC="tts/samples/xxx.wav"     # 音频或视频都行
ffprobe -v error -show_entries format=duration -show_entries stream=codec_name,channels,sample_rate \
  -of default=noprint_wrappers=1 "$SRC"
# 若是视频：先抽音轨
# ffmpeg -y -i input.mp4 -vn -ac 1 -ar 16000 /tmp/src.wav   # 然后 SRC=/tmp/src.wav
```
> Whisper 内部按 16kHz 单声道处理；提前 `-ar 16000 -ac 1` 转一下最省事（非必须，whisper 会自动重采样）。

### Step 1 — 纯文本转写（最常用）
```bash
PYTHONPATH=/Users/boom/Demo/AINews tts/.venv/bin/python - <<'PY'
import whisper
m = whisper.load_model("small")              # 默认档；要精确换 "medium"
r = m.transcribe(
    "tts/samples/xxx.wav",
    language="zh", fp16=False,
    initial_prompt="以下是简体中文普通话内容。涉及公司：Anthropic、OpenAI、谷歌、英伟达。",
)
print(r["text"].strip())
PY
```

### Step 2 —（可选）导出带时间戳 / SRT 字幕
```bash
PYTHONPATH=/Users/boom/Demo/AINews tts/.venv/bin/python - <<'PY'
import whisper
from whisper.utils import get_writer
m = whisper.load_model("small")
r = m.transcribe("tts/samples/xxx.wav", language="zh", fp16=False,
                 initial_prompt="以下是简体中文普通话内容。")
# 段级时间戳
for seg in r["segments"]:
    print(f'[{seg["start"]:6.2f}-{seg["end"]:6.2f}] {seg["text"].strip()}')
# 一键导出 .srt / .txt / .vtt / .tsv / .json 到指定目录
get_writer("srt", "tts/samples")(r, "xxx.wav")   # -> tts/samples/xxx.srt
PY
```

### Step 3 —（可选）词级时间戳（做精切 / 句末边界对齐）
```bash
PYTHONPATH=/Users/boom/Demo/AINews tts/.venv/bin/python - <<'PY'
import whisper
m = whisper.load_model("medium")             # 词级时间戳建议 medium 起步
r = m.transcribe("tts/samples/xxx.wav", language="zh", fp16=False, word_timestamps=True)
for seg in r["segments"]:
    for w in seg.get("words", []):
        print(f'{w["start"]:6.2f}-{w["end"]:6.2f} {w["word"]}')
PY
```
> 这一步与 `从视频中分离人声指南.md` Step 5 同源：分离出的人声做 zero_shot 切片时，就靠词级时间戳找句末边界。

### Step 4 — 硬验收（别省）
- 抽查**关键词 / 专有名词 / 数字**是否命中，句子是否通顺。
- 出现中英混杂乱码、人名整段错、突然繁体 ⇒ 没转对：① 加/改 `initial_prompt`；② 升档 `small→medium`；③ 长音频复读 ⇒ `condition_on_previous_text=False`。

## 5. 🔌 扩展口（当前单引擎，后续在此接入新模型）

> 现在只有 openai-whisper。**未来接 faster-whisper / FunASR-SenseVoice / 云端 API 时，遵循下面这套与 TTS 侧 `providers/` 同构的契约，业务/pipeline 代码只依赖契约、不依赖具体引擎。**

### 5.1 统一契约（建议落成 `tts/asr/base.py`）

```python
from dataclasses import dataclass, field
from typing import Optional, List
from abc import ABC, abstractmethod

@dataclass
class ASRRequest:
    audio_path: str                      # 输入音频/视频路径
    language: str = "zh"                 # 语种
    model: str = "small"                 # 档位/模型名（引擎自解释）
    initial_prompt: Optional[str] = None # 领域提示词（专名/简体）
    word_timestamps: bool = False        # 是否要词级时间戳
    extra: dict = field(default_factory=dict)  # 引擎私有参数

@dataclass
class ASRSegment:
    start: float
    end: float
    text: str
    words: Optional[List[dict]] = None   # [{start,end,word}] when available

@dataclass
class ASRResult:
    text: str                            # 全文纯文本
    segments: List[ASRSegment]           # 段级（含可选词级）
    status: str                          # "ok" | "error" | "unavailable"
    backend: str                         # 实际使用的引擎名
    error: Optional[str] = None
    meta: dict = field(default_factory=dict)  # 如 {rtf, model, device}
    @property
    def ok(self) -> bool: return self.status == "ok"

class ASRBackend(ABC):
    """所有 ASR 引擎实现这个接口。换引擎不动业务代码。"""
    name: str = "base"
    def is_available(self):              # (usable: bool, reason: str)，不抛异常
        return True, "ok"
    @abstractmethod
    def transcribe(self, req: ASRRequest) -> ASRResult:
        """audio -> text。永不抛异常，统一返回 ASRResult。"""
        raise NotImplementedError
```

> 这套与 `tts/providers/base.py`（`TTSRequest`/`TTSResult`/`TTSProvider`）刻意保持一致：Request/Result dataclass + 抽象 Backend + `is_available()` 探测 + 「永不抛、返回 Result」。新引擎实现后在 `tts/asr/__init__.py` 注册即可（仿 `tts/providers/__init__.py`）。

### 5.2 当前唯一实现：`WhisperBackend`（行为锚点）
- `name = "openai-whisper"`
- 内部即第 4 节那套：`whisper.load_model(model)` + `transcribe(..., language, fp16=False, initial_prompt, word_timestamps)`。
- `is_available()`：`import whisper` 成功且 `~/.cache/whisper/<model>.pt` 在/可下。

### 5.3 候选扩展引擎对照（**下面数字是公开经验值/预期，非本机实测；接入时务必按第 2 节同法重测填表**）

| 引擎 | 设备 | 相对速度 | 中文质量 | 何时选它 |
|---|---|---|---|---|
| **openai-whisper**（现役） | CPU | 基准 | small◯ / medium✓ | 默认；本地、无依赖洁癖 |
| **faster-whisper**（CTranslate2） | CPU/GPU | 预期 ~4× 同档 whisper、内存更省 | 与 whisper 同模型同质量 | **要快/要 GPU/批量长音频**的首选扩展 |
| **FunASR / SenseVoice-Small** | CPU/GPU | 预期快、轻 | 中文/带标点/情感，专为中文优化 | 纯中文、要标点/ITN、要说话人 |
| **whisper.cpp** | CPU(量化) | 预期快、超省内存 | 同 whisper 模型 | 极致轻量/无 torch 环境 |
| **云端 API**（如各家 ASR） | 远端 | 取决网络 | 通常很好 | 不想本地养模型、可接受外发音频（注意隐私/版权） |

> ⚠️ 接云端 = 把音频外发，受版权/隐私约束（见 `从视频中分离人声指南.md` 第 5 节版权约定）。默认本地优先。

## 6. ⚠️ 踩坑清单

1. **CPU 没设 `fp16=False`** → 一堆 `FP16 is not supported on CPU` 警告且更慢。CPU 永远 `fp16=False`。
2. **不给 `language`** → whisper 自己猜语种，中文短音频容易猜成英/日，全程跑偏。中文固定 `language="zh"`。
3. **不给 `initial_prompt`** → 公司/人名乱转（Anthropic→「And for a pic」）、简繁混杂、没标点。一句提示词解决大半（第 2 节横评已证）。
4. **盲目上 `large`** → CPU 上又慢又吃内存，多数场景 `small`+prompt 或 `medium` 足够。
5. **本机开 MPS 想加速** → 反而更慢（实测）。openai-whisper 在 Apple Silicon 上用 CPU。
6. **长音频「复读机」重复** → 设 `condition_on_previous_text=False`；或切段转写再拼。
7. **验收只看「有输出」** → 必须抽查专名/关键词/通顺度，乱码就是没转对（与分离指南坑 7 同理）。
