下面进入详细内容。

第一条，OpenAI 称「聊天已死」，ChatGPT 将重写为 Agent 超级应用。
OpenAI 内部称「聊天已死」，启动 ChatGPT 上线以来最大改版，将其重写为以 Agent 为核心的「超级应用」。
为什么重要：看点：头部实验室第一次公开宣判「对话框」范式终结——竞争主轴从「谁答得好」转向「谁能替你把事办完」，也是 OpenAI 给 IPO 投资人「ChatGPT 不止是聊天玩具」的答案。
具体影响包括：
一，产品形态：聊天框让位给 Agent 工作流，编程、生图、第三方应用被编排进同一入口。
二，竞争格局：与 Anthropic（Claude Code）、谷歌（Gemini Agent）的「Agent 平台」之争全面摊牌。
三，商业逻辑：押注企业付费撑起 IPO 估值叙事，消费级「答问」不再是核心增长点。
信息来源：金融时报，TechCrunch，The Decoder。
可信度：中高。
待验证信息：新版界面与「超级应用」尚未上线（官方称未来数周推出），合作方清单与 Atlas 整合细节待落地。

第二条，Perplexity 推「搜索即代码」，让模型自己写检索管线。
Perplexity 发布「Search as Code」，让 AI Agent 自行生成 Python 脚本来调度搜索，取代固定的搜索 API。
为什么重要：看点：检索从「调用固定 API」升级为「模型现写现跑代码」，是 Agent 把外部工具内化为可编程能力的方向标，也直指 RAG/搜索接口的下一代形态。
具体影响包括：
一，开发者：复杂检索任务可显著省 token、降成本，长链路任务不易「跑偏」。
二，搜索/RAG 厂商：固定 API 范式受挑战，「让模型自写管线」可能成新基线。
三，安全：模型自动执行代码须依赖沙箱隔离，注入与越权风险成新焦点。
信息来源：Perplexity Research，The Decoder，Winbuzzer。
可信度：高。
待验证信息：85% 省 token 为官方单任务数据，跨场景表现与沙箱安全性需第三方验证。

第三条，ChatGPT 上线「锁定模式」，断网防提示注入窃密。
OpenAI 为 ChatGPT 推出可选的「锁定模式（Lockdown Mode）」，通过限制对外网络请求来降低提示注入导致的数据外泄风险。
为什么重要：看点：当 Agent 被允许联网与操作工具，提示注入就从「理论风险」变成「会偷数据的真攻击」——头部厂商把「主动断网」做成产品开关，等于承认现有防护挡不住，先给高敏用户兜底。
具体影响包括：
一，企业：处理敏感数据的团队多了一个「以功能换安全」的合规开关。
二，行业：把提示注入风险产品化、显性化，或推动同行跟进类似「安全档位」。
三，局限：官方承认锁定模式不能根治注入，安全责任仍在使用方流程设计。
信息来源：OpenAI，TechCrunch，The Decoder。
可信度：高。
待验证信息：锁定模式实际防护强度与对正常工作流的影响，需安全团队实测评估。
