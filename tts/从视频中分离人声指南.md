# 从视频/音频中分离人声 —— Agent 操作指南（踩坑 + 成功路径）

> 读者：后续接手的 AI agent。
> 目的：从一段**带背景音乐的视频/录音**里，分离出**干净人声**，并（可选）切成可直接喂给 CosyVoice zero_shot 的 prompt。
> 原则：**照着「成功路径」走，先看「踩坑清单」避免重蹈覆辙。**本文命令均在仓库根 `/Users/boom/Demo/AINews` 下执行，Python 一律用隔离环境 `tts/.venv/bin/python`。

---

## 0. 结论先行（TL;DR）

1. **不要靠耳朵猜有没有 BGM**，用客观指标判断（HPSS 节拍 + 静音检测）。
2. **有背景音乐 → 必须 demucs 源分离**。`ffmpeg` 轻降噪只能去稳态噪声，**去不掉音乐**。
3. **别对整段长音频直接跑 demucs**（CPU 极慢）。先 `ffmpeg` 切一段语音密集的 chunk 再分离。
4. **轻处理**：分离 → 单声道 + 24kHz + 70Hz 高通就够。**禁止**降噪过度/混响/变声。
5. **验收必须用 whisper 转写产物对照**，别只看「没报错 / 有音频文件」。
6. 若做 zero_shot 克隆：**prompt 必须短（6–10s）+ transcript 精确**；切片要卡在**句末边界**，否则断音。

## 2. ⚠️ 踩坑清单（每条都真实踩过，务必避开）

### 坑 1：靠猜判断有没有背景音乐
- **现象**：录音听起来"还行"，就以为是干净人声，直接拿去用。
- **真相**：本次源是 bilibili 录音，**底下压着一条带节拍的音乐床**。
- **正确做法**：客观测。
  - HPSS：打击乐能量占比高（>40%）+ 稳定 tempo（60–140bpm）⇒ 有音乐床。
  - 静音检测：`silencedetect` 在语音段**几乎找不到静音间隙**、动态范围被压窄（<12dB）⇒ 有连续底噪/音乐（或被重度限幅）。
  - 实测本例：percussive ≈ 46–50%，tempo 110–134bpm，0–120s 无静音 ⇒ **确诊有 BGM**。

### 坑 2：以为"轻度降噪"能去掉音乐
- **真相**：`ffmpeg afftdn`/`highpass` 等只削稳态噪声/低频，**对音乐无能为力**。
- **正确做法**：有音乐 = 必须 **demucs 源分离**（`--two-stems=vocals`）。

### 坑 3：对整段长音频直接 demucs（CPU 巨慢）
- **现象**：533s 全篇在 CPU 上跑 htdemucs，慢到无法接受。
- **正确做法**：**先 `ffmpeg` 切一段 ~90–120s 语音密集 chunk，再对 chunk 分离**。够取素材即可。

### 坑 4：乱切导致"断音"（最隐蔽）
- **现象**：随手 `ffmpeg -ss X -t Y` 切出来的 prompt，开头/结尾有突变、像被"咔"一下切断。
- **根因**：①切到了下一个字的**起音**或半个词；②demucs 分离后的人声**非常连续，几乎没有静音间隙**，没有天然安全的切点。
  - 本例反面教材：6.6s prompt 切到 `[10.40, 17.00]`，而 17.00s 正好是下一个字"这"的起音 ⇒ 结尾断音。
- **正确做法**（见成功路径 Step 5）：
  1. 用 whisper **词级时间戳**（`word_timestamps=True`）找**句末边界**（标点/语义停顿处）；
  2. 切点用 librosa 在边界 ±150ms 内**对齐到能量最低点（RMS 谷）**；
  3. 边缘加**极短防爆音淡变**（淡入 12ms / 淡出 30ms）——只防 click，**不改音色**。

### 坑 5：长任务丢后台就提前"完成"
- **现象**：把下载/分离丢进后台 `&` 后，子 agent 立刻返回"已完成"，其实进程还在跑。
- **正确做法**：长任务要么**等真正结束再返回**；要么用 `until ! pgrep -f "<cmd>"; do sleep 15; done` 这类循环**等到进程退出**再核对产物。别把"已启动"当"已完成"。

### 坑 6：模型/权重下载慢且会断（modelscope）
- **现象**：multi-GB 下载中途 `FileDownloadError`，关键文件缺失。
- **正确做法**：**断点续传 + 重试循环**，且**只拉必要文件**。例如 CosyVoice2 在 `load_jit=False,load_trt=False` 下只需 `llm.pt`/`flow.pt`/`hift.pt`/`campplus.onnx`/`speech_tokenizer_v2.onnx`，那些 `.zip`/`.batch.onnx`（JIT/TRT 用）不必下。

### 坑 7：验收只看"没报错"
- **现象**：pipeline 全程 0 报错、`status=ok`、有 wav 文件，**实际音频是乱码**（whisper 转写全是无关字）。
- **正确做法**：**硬验收闸门 = 用 whisper 转写产物，逐条对照原稿**，关键词都在、语句通顺才算 PASS。别让"无异常"冒充"对"。

### 坑 8：gitignore 把 config 引用的 prompt 挡在库外
- **现象**：`tts/assets/*.wav` 被忽略，导致 config 指向的 prompt 没入库；而子目录 `prompt_candidates/*.wav` 又"漏网"被误暂存。
- **正确做法**：克隆参考音频（含版权来源/可重生成）默认不入库；**唯独 config 实际引用的那个正式 prompt 用 `!` 例外放行**保证自洽；子目录单独加忽略规则防漏网。

### 过度处理（红线）
- **禁止**：重度降噪、加混响、变调/变声、整体 loudnorm 改动态。这些会污染音色，破坏克隆保真度。
- **只允许**：源分离 + 单声道/重采样 + 70Hz 高通（去低频隆隆声） + 边缘极短淡变（防 click）。峰值若过 0dB 才做轻微限幅。

### Step 5 —（可选）切 zero_shot 克隆 prompt：句末边界 + 能量谷 + 防爆音淡变
> demucs 人声很连续、几乎无静音，**不能随手切**，否则断音（坑 4）。

```bash
# 5a 取词级时间戳，定位句末边界（标点/停顿）
PYTHONPATH=/Users/boom/Demo/AINews tts/.venv/bin/python - <<'PY'
import whisper
r=whisper.load_model("medium").transcribe("tts/assets/<归档名>.wav",language="zh",fp16=False,word_timestamps=True)
for seg in r["segments"]:
    for w in seg.get("words",[]):
        mark="  <== 句末候选" if any(p in w["word"] for p in "。？！，") else ""
        print(f'{w["start"]:6.2f}-{w["end"]:6.2f} {w["word"]}{mark}')
PY
```
```python
# 5b 按"句末边界对"生成 6-10s 备选；切点对齐能量谷 + 12ms/30ms 淡变
# 存 /tmp/cut_cands.py 跑：PYTHONPATH=/Users/boom/Demo/AINews tts/.venv/bin/python /tmp/cut_cands.py
import librosa, soundfile as sf, numpy as np, os
y,sr=librosa.load("tts/assets/<归档名>.wav",sr=24000)
def snap(t,half=0.15):                      # 在 t±150ms 内找 RMS 最小处当切点
    if t<=0.05: return 0.0
    a=max(0,int((t-half)*sr)); b=min(len(y),int((t+half)*sr)); win=int(0.02*sr)
    if b-a<=win: return t
    seg=y[a:b]; rms=np.sqrt(np.convolve(seg**2,np.ones(win)/win,'valid'))
    return (a+int(np.argmin(rms))+win//2)/sr
BOUNDS=[...]                                  # 填 5a 得到的句末秒，如 0,1.94,4.84,10.52,...
pairs=[(s,e) for i,s in enumerate(BOUNDS) for e in BOUNDS[i+1:] if 6<=e-s<=10.5]
os.makedirs("tts/assets/prompt_candidates",exist_ok=True)
for k,(s,e) in enumerate(pairs,1):
    ss,ee=snap(s),snap(e); sub=y[int(ss*sr):int(ee*sr)].astype(np.float32).copy()
    fi,fo=int(0.012*sr),int(0.030*sr)
    if len(sub)>fi+fo: sub[:fi]*=np.linspace(0,1,fi,dtype=np.float32); sub[-fo:]*=np.linspace(1,0,fo,dtype=np.float32)
    out=f"tts/assets/prompt_candidates/c{k}_{ss:.2f}-{ee:.2f}.wav"; sf.write(out,sub,sr,subtype="PCM_16")
    print(out,f"dur={len(sub)/sr:.2f}s")
```
- 客观挑最干净的：测每段**首/尾 RMS / 全段 RMS**，两端比值越接近 0（落静音）越不会断音。再交人耳从中定。
- 选定后：把它复制成正式 prompt（如 `cp <选中> tts/assets/ref_my_voice_zh_prompt.wav`），并用 whisper(medium) 取**该短片的精确 transcript** 当 `prompt_text`（坑 7 同理，transcript 要贴合音频）。

## 4. 关键参数速查

| 项 | 值 | 说明 |
|---|---|---|
| 分离工具 | `demucs --two-stems=vocals`（htdemucs） | 去音乐床唯一靠谱手段 |
| 分离输入 | 先切 ~90–120s chunk | 全篇 CPU 太慢（坑 3） |
| 归档采样率/声道 | 24000Hz / 单声道 | CosyVoice2 内部按 16k 取，24k 归档更稳 |
| 处理滤波 | 仅 `highpass=f=70` | 去低频；不降噪/不混响/不变声 |
| zero_shot prompt 时长 | **6–10s** | 过长（如 30s）会乱码（坑 4 + TTS 侧） |
| 切点对齐 | whisper 词级时间戳 + RMS 谷 | 防断音（坑 4） |
| 防爆音淡变 | 淡入 12ms / 淡出 30ms | 只防 click，不动音色 |
| 验收 | whisper 转写对照原稿 | 唯一可信闸门（坑 7） |
